# 积分记录与库存监控系统 - 技术文档

## 1. 项目简介
本项目是一个运行在树莓派上的轻量级 Web 管理系统，主要用于接收青龙面板自动化脚本上报的积分数据，并管理商品库存图片。

### 1.1 技术栈
- **后端**: Python 3.9+ / FastAPI
- **数据库**: SQLite
- **前端**: HTML5 / Bootstrap 5 / jQuery
- **部署**: Raspberry Pi OS / Systemd

## 2. 核心功能
1. **积分监控**: 
   - 记录 多个微信号在近百个小程序中的积分数据。
   - 提供按账号、小程序的积分统计与趋势图表。
2. **库存管理**:
   - 维护商品信息，支持图片上传与预览（模态框大图浏览）。
   - **排序与筛选**：支持按商品积分升序排列（客户端即时排序），支持商品名称搜索。
   - **管理操作**：支持小程序维度的删除、详情跳转（含积分排序参数）。
   - 记录库存变动历史。
3. **数据对接**:
   - 提供 RESTful API 接收青龙脚本的数据上报 (Auth: Bearer Token)。

## 3. API 接口设计

### 3.1 积分上报
`POST /api/v1/qinglong/report`
- **Headers**: `Authorization: Bearer {token}`
- **Body**:
```json
{
    "script_id": "xxx_sign",
    "data": {
        "wechat_accounts": [
            {
                "wechat_id": "wx_001",
                "points_data": [
                    {"program_id": "p_001", "current_points": 100}
                ]
            }
        ]
    }
}
```

> **用户身份识别逻辑说明**：
> 1. **手机号检测**：若 `wechat_id` 符合手机号格式（11位数字），系统会自动检查其是否已绑定微信账号。
> 2. **自动关联昵称**：
>    - 若手机号已绑定微信账号，系统将忽略请求体中的 `nickname`，强制使用已绑定微信账号的昵称。
>    - 若手机号未绑定微信账号，系统将强制设置昵称为空字符串（""）。
> 3. **兼容性**：非手机号格式的 `wechat_id` 将保持原有逻辑，优先使用请求体中的 `nickname`。

### 3.2 库存上报
`POST /api/v1/qinglong/stock-report`
- **Body**:
```json
{
    "program_id": "p_001",
    "products": [
        {"product_id": "prod_001", "stock": 50, "image_url": "http://..."}
    ]
}
```

### 3.3 图片上传
`POST /api/v1/upload/image`
- **Content-Type**: `multipart/form-data`
- **Params**: `file` (文件), `product_id` (可选), `program_id` (可选)
- **Response**:
```json
{
    "filename": "uuid.jpg",
    "path": "/static/uploads/uuid.jpg"
}
```

## 3.4 前端页面路由
- `GET /` : 仪表盘 (Dashboard)
- `GET /points` : 积分列表
- `GET /stock` : 库存管理

## 4. 数据库设计 (SQLite)

### 4.1 核心表结构
```sql
-- 1. 微信号表
CREATE TABLE wechat_accounts (
    id INTEGER PRIMARY KEY,
    wechat_id VARCHAR(50) UNIQUE NOT NULL,
    nickname VARCHAR(100)
);

-- 2. 小程序表
CREATE TABLE mini_programs (
    id INTEGER PRIMARY KEY,
    program_id VARCHAR(100) UNIQUE NOT NULL,
    program_name VARCHAR(200)
);

-- 3. 积分记录表
CREATE TABLE points_history (
    id INTEGER PRIMARY KEY,
    wechat_id VARCHAR(50),
    program_id VARCHAR(100),
    points INTEGER,
    report_time DATETIME,
    batch_id VARCHAR(50), -- 用于去重
    FOREIGN KEY(wechat_id) REFERENCES wechat_accounts(wechat_id),
    FOREIGN KEY(program_id) REFERENCES mini_programs(program_id)
);

-- 4. 商品表
CREATE TABLE products (
    id INTEGER PRIMARY KEY,
    program_id VARCHAR(100),
    product_id VARCHAR(100),
    product_name VARCHAR(200),
    image_local_path TEXT,
    stock INTEGER DEFAULT 0,
    UNIQUE(program_id, product_id)
);

-- 5. 库存历史
CREATE TABLE stock_history (
    id INTEGER PRIMARY KEY,
    program_id VARCHAR(100),
    product_id VARCHAR(100),
    old_stock INTEGER,
    new_stock INTEGER,
    change_time DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## 6. 部署指南 (树莓派)

1. **环境准备**
   ```bash
   sudo apt update
   sudo apt install python3 python3-pip libjpeg-dev zlib1g-dev
   ```

2. **安装项目**
   ```bash
   mkdir wechat-points-stock && cd wechat-points-stock
   python3 -m venv venv
   source venv/bin/activate
   pip install -r requirements.txt
   ```

3. **配置与启动**
   - 复制 `.env.example` 为 `.env` 并配置 `API_TOKEN`。
   - 初始化数据库: `python scripts/init_db.py`
   - 启动服务: `bash start.sh`

4. **Systemd 守护进程**
   - 创建 `/etc/systemd/system/points-stock.service` 指向 `start.sh`，实现开机自启。
