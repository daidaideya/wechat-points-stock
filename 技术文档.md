# 积分记录与库存监控系统 - 技术文档

## 1. 项目简介

本项目是一个轻量级 Web 管理系统，主要用于：

1. 接收青龙面板脚本上报的积分 / 现金 / 库存数据
2. 管理微信账号与小程序列表
3. 同步青龙定时任务状态
4. 通过 Bark 推送未上报提醒
5. 提供数据库备份恢复

### 1.1 技术栈

- **后端**: Python 3.11+ / FastAPI / SQLAlchemy / SQLite
- **前端**: Vue 3 / Vite / Element Plus
- **部署**: 本地 uvicorn / Docker Compose / systemd

## 2. 核心功能

1. **积分与现金监控**
   - 支持只报积分、只报现金、双报
   - 账号维度汇总积分 / 现金与今日变化
2. **小程序管理**
   - 收藏 / 归档 / 标签 / 备注
   - 青龙启停状态、cron 表达式展示
   - 按 cron 最早执行时间排序
3. **库存管理**
   - 商品库存与图片
   - 可兑换商品高亮（按最高用户积分）
   - 今日上下架变动（默认折叠）
4. **青龙联动**
   - OpenAPI 同步定时任务 enable/disable + schedule
5. **Bark 推送**
   - 每日定时推送未上报活跃小程序
6. **系统运维**
   - 访问密钥保护
   - 日志清理
   - SQLite 备份导出 / 导入恢复

## 3. API 接口设计

### 3.1 积分 / 现金上报

`POST /api/v1/qinglong/report`

- **Headers**: `Authorization: Bearer {token}`
- **Body**:

```json
{
  "script_id": "xxx_sign",
  "execution_time": "2026-07-16 20:00:00",
  "data": {
    "wechat_accounts": [
      {
        "wechat_id": "wx_001",
        "nickname": "可选",
        "points_data": [
          {
            "program_id": "p_001",
            "program_name": "示例小程序",
            "current_points": 100,
            "current_cash": 1.25
          }
        ]
      }
    ]
  }
}
```

规则：

- `current_points` / `current_cash` 均为可选 float
- **至少提供一个**
- 老脚本只传 `current_points` 完全兼容
- 未传维度入库为 `NULL`

> **用户身份识别逻辑说明**：
> 1. **手机号检测**：若 `wechat_id` 符合手机号格式（11位数字），系统会自动检查其是否已绑定微信账号。
> 2. **自动关联昵称**：
>    - 若手机号已绑定微信账号，系统将忽略请求体中的 `nickname`，强制使用已绑定微信账号的昵称。
>    - 若手机号未绑定微信账号，系统将强制设置昵称为空字符串（""）。
> 3. **兼容性**：非手机号格式的 `wechat_id` 将保持原有逻辑，优先使用请求体中的 `nickname`。

### 3.2 库存上报

`POST /api/v1/stock-report`

```json
{
  "program_id": "p_001",
  "products": [
    {
      "product_id": "prod_001",
      "product_name": "商品A",
      "stock": 50,
      "points": 100,
      "image_url": "http://..."
    }
  ]
}
```

### 3.3 图片上传

`POST /api/v1/upload/image`

- `multipart/form-data`
- 返回 `/static/uploads/...`

### 3.4 青龙 OpenAPI 同步

- `GET /api/v1/settings/qinglong`
- `POST /api/v1/settings/qinglong`
- `POST /api/v1/settings/qinglong/sync`

同步字段写入 `mini_programs`：

- `ql_cron_id`
- `ql_cron_name`
- `ql_is_disabled`（0 启用 / 1 禁用 / NULL 未匹配）
- `ql_schedule`
- `ql_command`
- `ql_matched_at`

### 3.5 Bark 推送

- `GET /api/v1/settings/bark`
- `POST /api/v1/settings/bark`
- `POST /api/v1/settings/bark/test`

配置项：

- `bark_enabled`
- `bark_server`（默认 `https://api.day.app`）
- `bark_device_key`
- `bark_push_time`（本地 `HH:MM`，默认 `20:00`）

服务启动后后台线程每分钟检查是否到达推送时间。

### 3.6 数据库备份

- `GET /api/v1/settings/database/export`
- `POST /api/v1/settings/database/import`

导入前会在同目录生成 `.pre_restore` 回滚副本。

### 3.7 前端页面路由（SPA `/app/`）

- `/dashboard` 仪表盘
- `/programs` 小程序列表
- `/users` 用户管理
- `/points` 积分总览
- `/stock` 库存管理
- `/settings` 系统设置

## 4. 数据库设计 (SQLite)

### 4.1 关键表与字段

```sql
-- 微信号
CREATE TABLE wechat_accounts (
  id INTEGER PRIMARY KEY,
  wechat_id VARCHAR(50) UNIQUE NOT NULL,
  nickname VARCHAR(100),
  device VARCHAR(100),
  phone VARCHAR(20),
  sort_order INTEGER DEFAULT 0
);

-- 小程序
CREATE TABLE mini_programs (
  id INTEGER PRIMARY KEY,
  program_id VARCHAR(100) UNIQUE NOT NULL,
  program_name VARCHAR(200),
  auth_type VARCHAR(20) DEFAULT 'code',
  sort_order INTEGER DEFAULT 0,
  is_favorite INTEGER DEFAULT 0,
  note TEXT,
  tags TEXT,
  is_archived INTEGER DEFAULT 0,
  archived_at DATETIME,
  ql_cron_id INTEGER,
  ql_cron_name VARCHAR(200),
  ql_is_disabled INTEGER,
  ql_matched_at DATETIME,
  ql_command VARCHAR(500),
  ql_schedule VARCHAR(100)
);

-- 积分 / 现金历史
CREATE TABLE points_history (
  id INTEGER PRIMARY KEY,
  wechat_id VARCHAR(50),
  program_id VARCHAR(100),
  points FLOAT,          -- 可空
  cash FLOAT,            -- 可空
  report_time DATETIME,
  batch_id VARCHAR(50)
);

-- 系统设置（节选）
CREATE TABLE system_settings (
  id INTEGER PRIMARY KEY,
  max_log_entries INTEGER DEFAULT 10000,
  max_retention_days INTEGER DEFAULT 30,
  access_protection_enabled INTEGER DEFAULT 0,
  access_key VARCHAR(255),
  ql_base_url VARCHAR(255),
  ql_client_id VARCHAR(100),
  ql_client_secret VARCHAR(255),
  ql_last_sync_at DATETIME,
  ql_last_sync_status VARCHAR(255),
  bark_enabled INTEGER DEFAULT 0,
  bark_server VARCHAR(255),
  bark_device_key VARCHAR(255),
  bark_push_time VARCHAR(16),
  bark_last_push_at DATETIME,
  bark_last_push_status VARCHAR(255),
  updated_at DATETIME
);
```

> 无 Alembic。新增可空列通过 `ensure_*_columns` 懒迁移。

## 6. 前端交互要点

### 6.1 小程序列表

- 指标显示策略：
  - 无库存：隐藏库存指标块
  - 无现金数据：隐藏现金指标块
  - 无积分：不显示正积分块
- 操作按钮放在卡片右下角
- 详情 / 库存都是弹窗
- 备注使用 `white-space: pre-wrap` 保留换行

### 6.2 移动端导航

- 不使用 Element Plus `el-drawer` 作为主导航
- `App.vue` 使用自定义 `.mobile-nav-shell`
- 桌面 ↔ 手机断点切换时自动关闭侧栏，避免遮罩残留导致按钮点不动

## 7. 部署注意

1. 生产启动前必须 `npm run build`
2. 后端托管 `frontend/dist`
3. `.env` 至少配置 `API_TOKEN`
4. Bark / 青龙密钥保存在数据库设置表中，不要写入代码仓库
